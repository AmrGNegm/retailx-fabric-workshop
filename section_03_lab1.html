🎯 Your Task — Code Challenge
Open the Advanced Editor in DFG2 and write an M query that: ① Reads the CSV from the GitHub URL ② Promotes the first row as headers ③ Renames ALL 21 columns to snake_case ④ Casts the correct data types (dates as date, numbers as number, integers as Int64) This is the Bronze ingestion contract — the output of this query lands in raw_sales.
💪 Try it yourself first. Open the Advanced Editor (Home → Advanced Editor) and write the M code from scratch using the structure hint below. The column names to rename are: Row ID, Order ID, Order Date, Ship Date, Ship Mode, Customer ID, Customer Name, Segment, Country, City, State, Postal Code, Region, Product ID, Category, Sub-Category, Product Name, Sales, Quantity, Discount, Profit.
💡 Hint — M code skeleton (fill in the blanks)
The M query has 4 steps: Source → Headers → Renamed → TypedCols. Fill in the blanks marked with _____ below. Use the Advanced Editor (Home → Advanced Editor) to write your answer.
let Source = Csv.Document(Web.Contents( "_____URL HERE_____"), [Delimiter=",", Encoding=65001, QuoteStyle=QuoteStyle.None]), Headers = Table.PromoteHeaders(_____, [PromoteAllScalars=true]), Renamed = Table.RenameColumns(Headers, { {"Row ID", "_____"}, // snake_case version {"Order ID", "_____"}, {"Order Date", "_____"}, {"Ship Date", "_____"}, {"Ship Mode", "_____"}, // ... continue for all 21 columns ... {"Sales", "_____"}, {"Quantity", "_____"}, {"Discount", "_____"}, {"Profit", "_____"} }), TypedCols = Table.TransformColumnTypes(Renamed, { {"row_id", _____}, // integer type {"order_date", _____}, // date type {"ship_date", _____}, // date type {"sales", _____}, // number type {"quantity", _____}, // integer type {"discount", _____}, // number type {"profit", _____} // number type }) in TypedCols

✅ Full M Code Answer Paste into Advanced Editor → Home → Advanced Editor
DFG2-Sales-CSV — Power Query M (full solution) 
let
  Source    = Csv.Document(Web.Contents(
                "https://raw.githubusercontent.com/plotly/datasets/master/superstore.csv"),
                [Delimiter=",", Encoding=65001, QuoteStyle=QuoteStyle.None]),

  Headers   = Table.PromoteHeaders(Source, [PromoteAllScalars=true]),

  Renamed   = Table.RenameColumns(Headers, {
    {"Row ID",        "row_id"},
    {"Order ID",      "order_id"},
    {"Order Date",    "order_date"},
    {"Ship Date",     "ship_date"},
    {"Ship Mode",     "ship_mode"},
    {"Customer ID",   "customer_id"},
    {"Customer Name", "customer_name"},
    {"Segment",       "segment"},
    {"Country",       "country"},
    {"City",          "city"},
    {"State",         "state"},
    {"Postal Code",   "postal_code"},
    {"Region",        "region"},
    {"Product ID",    "product_id"},
    {"Category",      "category"},
    {"Sub-Category",  "sub_category"},
    {"Product Name",  "product_name"},
    {"Sales",         "sales"},
    {"Quantity",      "quantity"},
    {"Discount",      "discount"},
    {"Profit",        "profit"}
  }),

  TypedCols = Table.TransformColumnTypes(Renamed, {
    {"row_id",      Int64.Type},
    {"order_date",  type date},
    {"ship_date",   type date},
    {"sales",       type number},
    {"quantity",    Int64.Type},
    {"discount",    type number},
    {"profit",      type number}
  })

in TypedCols
Why this works:
• Csv.Document + Web.Contents — reads the raw file over HTTP without a gateway
• Table.PromoteHeaders — makes row 1 into column headers
• Table.RenameColumns — takes a list of {old, new} pairs — all 21 at once
• Table.TransformColumnTypes — only the 7 columns that need non-text types are listed; the rest stay as text
• in TypedCols — the final step name is the output of the query
Instructor Note
Give students 5–7 minutes to attempt this from scratch. The most common mistakes: forgetting the closing }) on RenameColumns, using wrong type syntax (type int64 instead of Int64.Type), and forgetting to reference "Source" in the Table.PromoteHeaders call. Students who finish early: ask them to also rename the query (right-click → Rename) to "raw_sales" in the Queries panel.

💡 Hint — Update Method Decision
The "+ Add data destination" button is at the bottom-right of the canvas — you may need to scroll down or look below the query preview. After selecting Lakehouse → LH_RetailX → New table → raw_sales, you will be asked for the update method: → Replace — drops and recreates the table on every run ← use this for Bronze in this workshop → Append — adds new rows without removing existing ones → (Upsert is not available in DFG2 natively — that requires notebook logic) For a Bronze table that is a full reload from a static CSV file, "Replace" is correct. In production with real streaming or incremental data, you would use "Append" or move to a notebook with Delta MERGE logic.
