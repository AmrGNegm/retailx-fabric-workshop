requests.get · raise_for_status · .json()
CODE Quiz
▼
🎯 Code Challenge 1
Write the code that calls the REST Countries API and loads the JSON response. The API is public, no authentication needed.
  • →Make an HTTP GET request to API_URL with a 30-second timeout
  • →Raise an error immediately if the HTTP status code is not 200
  • →Parse the JSON response body into a Python list
  • →Print how many countries were returned
  • →Expected output: Fetched 250 countries
💡 Hint — requests pattern
The requests library works like this: response = requests._____(url, timeout=_____) # HTTP method + timeout response._____() # check for HTTP errors data = response._____() # parse JSON body → Python list/dict Common methods: requests.get(url) → GET request response.raise_for_status() → raises HTTPError if status >= 400 response.json() → parses body as JSON → returns list or dict The API returns a JSON array: [{country1}, {country2}, ...] So response.json() returns a Python list of dicts.
✅ Challenge 1 AnswerCell 3
Cell 3 — API Fetch
# ── Fetch from REST Countries API ─────────────────────────────
print("Fetching REST Countries API...")

r = requests.get(API_URL, timeout=30)
r.raise_for_status()           # raises HTTPError if status >= 400
countries = r.json()           # parse JSON → list of 250 country dicts

print(f"Fetched {len(countries)} countries")
Why raise_for_status()? Without it, a 404 or 500 error would silently return an empty or error body, and the notebook would fail later with a confusing error. raise_for_status() fails fast with a clear HTTP error message.

Why timeout=30? Without a timeout, if the API is slow, the notebook hangs forever and blocks the pipeline. 30 seconds is generous for a public API returning 250 records.
Instructor Note
This step typically takes students 3–5 minutes. Common mistakes: using requests.post() instead of .get(), forgetting .raise_for_status(), and trying to use response.text instead of .json(). Run the cell together — the expected output is "Fetched 250 countries" (exact number varies slightly as the API adds/removes territories).

CODE Quiz

🎯 Code Challenge 2
Before parsing, save the raw API response as a JSON file in the Lakehouse Files zone. This is the immutable archive of the exact source data.
  • →Generate a timestamp string in format YYYYMMDD_HHMMSS
  • →Use mssparkutils.fs.put() to write the JSON to: Files/bronze/countries/countries_{ts}.json

🔑 Key concept: This is what makes the Bronze layer truly immutable. Even if the API changes or goes offline, you always have the original raw response to reprocess. Never skip this step.
💡 Hint — mssparkutils + json.dumps
Generating a timestamp: ts = datetime.now().strftime("_____") # format: YYYYMMDD_HHMMSS Writing a file to the Lakehouse: mssparkutils.fs.put( path, # string: "Files/bronze/countries/countries_{ts}.json" content, # string: must be a STRING, not a Python list/dict overwrite # bool: True ) Converting a Python list/dict to a JSON string: json_string = json._____(python_object) # not json.load() — that reads, this writes
✅ Challenge 2 AnswerCell 4
Cell 4 — Save Raw JSON
# ── Save raw JSON to Files/bronze/ (immutable archive) ────────
ts       = datetime.now().strftime("%Y%m%d_%H%M%S")
raw_path = f"Files/bronze/countries/countries_{ts}.json"

mssparkutils.fs.put(
    raw_path,
    json.dumps(countries),   # serialize list → JSON string
    overwrite=True
)
print(f"Saved raw JSON → {raw_path}")
Why a timestamp in the filename? Each run produces a uniquely named file. This preserves every API snapshot — if you run the pipeline daily, you have 365 files, one per day. You can see exactly what the API returned on any given date.

Why json.dumps() not json.dump()? dumps() returns a string (for writing to a variable or mssparkutils). dump() writes to a file object. mssparkutils.fs.put() expects a string — use dumps().

🎯 Code Challenge 3 — The hardest one
Write a parse(c) function that takes one country dict and returns a flat dict with 11 fields. Then apply it to all 250 countries and create a Spark DataFrame.
  • →The function must extract: country_code, name_common, region, subregion, population, area_km2, capital, currency_code, currency_name, latitude, longitude
  • →Handle: nested objects, optional keys, None values, empty lists
  • →Apply with a list comprehension: [parse(c) for c in countries]

⚠️ Tested & Fixed — Common traps in this function
c.get("subregion", "") returns None when the key exists but value is None — use c.get("subregion") or ""
int(c.get("population", 0)) crashes when value is None — use int(c.get("population") or 0)
Same applies to area_km2 — some countries have "area": null in the API response.
💡 Hint — parse() skeleton with nested access patterns
def parse(c): curs = c.get("currencies", {}) # dict of currency objects (may be empty) return { "country_code" : c.get("cca2", ""), "name_common" : c.get("name", {}).get("_____", ""), # nested! "region" : c.get("region", ""), "subregion" : c.get("subregion") or "", # None-safe! "population" : int(c.get("population") or _____), "area_km2" : float(c.get("area") or _____), "capital" : c.get("capital", [""])[0] if c.get("_____") else "", "currency_code": list(curs.keys())[0] if curs else "", "currency_name": list(curs.values())[0].get("_____","") if curs else "", "latitude" : float(c.get("latlng", [0,0])[_____]) if c.get("latlng") else 0.0, "longitude" : float(c.get("latlng", [0,0])[_____]) if c.get("latlng") else 0.0, } # Apply to all countries records = [parse(c) for c in countries] # Create Spark DataFrame df = spark.createDataFrame(_____)
API structure reference for one country: { "cca2": "SA", "name": {"common": "Saudi Arabia", "official": "..."}, "region": "Asia", "subregion": "Western Asia", ← can be null for some countries! "population": 35950396, ← can be null for tiny territories! "area": 2149690.0, ← can be null! "capital": ["Riyadh"], ← list of strings (empty list = no capital) "currencies": {"SAR": {"name": "Saudi riyal", "symbol": "..."}}, "latlng": [25.0, 45.0] ← [latitude, longitude] }
✅ Challenge 3 AnswerCell 5 — tested with 250 edge cases
Cell 5 — parse() function + DataFrame creation
# ── Parse JSON records → flat Python dicts ────────────────────
def parse(c):
    curs = c.get("currencies", {})
    return {
        "country_code" : c.get("cca2", ""),
        "name_common"  : c.get("name", {}).get("common", ""),
        "region"       : c.get("region", ""),
        "subregion"    : c.get("subregion") or "",      # None-safe
        "population"   : int(c.get("population") or 0), # None-safe
        "area_km2"     : float(c.get("area") or 0),     # None-safe
        "capital"      : c.get("capital", [""])[0] if c.get("capital") else "",
        "currency_code": list(curs.keys())[0] if curs else "",
        "currency_name": list(curs.values())[0].get("name", "") if curs else "",
        "latitude"     : float(c.get("latlng", [0,0])[0]) if c.get("latlng") else 0.0,
        "longitude"    : float(c.get("latlng", [0,0])[1]) if c.get("latlng") else 0.0,
    }

records = [parse(c) for c in countries]

# ── Create Spark DataFrame ─────────────────────────────────────
df = spark.createDataFrame(records)
print(f"DataFrame created: {df.count()} rows, {len(df.columns)} columns")
display(df.limit(3))
3 key None-safe patterns used here:
• c.get("subregion") or "" — handles both missing key AND key with None value
• int(c.get("population") or 0) — int(None) crashes; int(0) is fine
• c.get("capital",[""])[0] if c.get("capital") else "" — handles None AND empty list

Note on latlng: index 0 = latitude, index 1 = longitude. This is a common source of bugs — always verify which index is which.
Instructor Note
This is the hardest coding challenge in the whole workshop. Give students 8–10 minutes. Most will have the structure right but miss the None-safe patterns. When they hit errors, ask: "What happens when you call int(None)?" The or 0 / or "" pattern is a Pythonic idiom they should know. After this challenge, run display(df.limit(3)) together so everyone sees the parsed data.

💡 Hint — withColumn patterns
df = (df .withColumn("_ingestion_ts", F.___________()) # Spark timestamp function .withColumn("_source", F.lit(_____)) # literal string value .withColumn("_batch_id", F.lit(_____))) # literal string value # DQ Check — count bad rows and assert = 0 total = df.count() assert df.filter(F.col("_____") == "").count() == 0, "DQ: empty names found" assert df.filter(F.col("_____") == "").count() == 0, "DQ: empty codes found" print(f"[DQ] {total} records — all checks passed")
Spark functions to use: → F.current_timestamp() → Timestamp column with the current time → F.lit("some string") → Literal column — same value in every row Remember: all .withColumn() calls must be wrapped in outer () because Python does not allow multi-line chained method calls without them.
✅ Challenge 4 AnswerCell 6
Cell 6 — Audit columns + DQ checks
# ── Add audit columns ─────────────────────────────────────────
df = (df
    .withColumn("_ingestion_ts", F.current_timestamp())
    .withColumn("_source",       F.lit(API_URL))
    .withColumn("_batch_id",     F.lit(ts)))

# ── Data Quality Checks ────────────────────────────────────────
total = df.count()
assert df.filter(F.col("name_common")  == "").count() == 0, \
    "DQ FAILED: empty country names found"
assert df.filter(F.col("country_code") == "").count() == 0, \
    "DQ FAILED: empty country codes found"
print(f"[DQ] {total} records — all checks passed")
Why F.current_timestamp() instead of Python datetime.now()?
F.current_timestamp() runs at the Spark executor level and returns a TimestampType column. datetime.now() is a Python object — you would need F.lit(str(datetime.now())) which is less precise and stored as a string, not a proper timestamp type.

Why two DQ checks and not one? Each check tests a different thing. If only the name_common check existed and country_code was empty, the data would silently pass. Defensive DQ means checking every critical field independently.

💡 Hint — Delta write pattern
(_____ # start with opening paren .write .format("_____") # Delta format .mode("_____") # overwrite or append .option("overwriteSchema", "_____") # "true" as a string .saveAsTable(_____)) # use the parameter variable print(f"Written {_____}: {_____} rows") display(spark.table(_____).limit(5))
Important: → The entire chain MUST be wrapped in outer () for multi-line Python → saveAsTable() takes the table name as a string → Use RAW_TABLE (the variable) not "raw_countries" (hardcoded) → overwriteSchema must be the string "true", not the boolean True
✅ Challenge 5 AnswerCell 7
Cell 7 — Write to Delta table
# ── Write to Delta (Bronze layer) ─────────────────────────────
(df.write
   .format("delta")
   .mode("overwrite")
   .option("overwriteSchema", "true")
   .saveAsTable(RAW_TABLE))

print(f"Written {RAW_TABLE}: {df.count()} rows")
display(spark.table(RAW_TABLE).limit(5))
saveAsTable() vs save(path): saveAsTable("raw_countries") registers the table in the Lakehouse metastore, making it immediately queryable by name in SQL endpoint and other notebooks. .save(path) just writes Parquet files without registering — you would need a separate CTAS statement to query it.

overwriteSchema "true": Without this, if you add a column to the DataFrame between runs (e.g., a new audit column), Delta will reject the write because the schema changed. With it, Delta updates the schema and overwrites the data atomically.
Instructor Note
Common student mistake: forgetting the outer parentheses → IndentationError on the second line. Point this out explicitly — it is a Python syntax rule, not a Spark rule. The chain works in one line without parens but becomes unreadable. When the write succeeds, switch to LH_RetailX to show the raw_countries table appearing in the Tables section.
